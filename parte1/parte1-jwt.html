JWT - JSON Web Token

Padrão para criação de tokens que asseguram acesso a recursos remotos. Para garantir integridade, tokens JWT podem ser assinados digitalmente, por exemplo com o algoritmo HMAC, ou um par de chaves pública/privada usando RSA. Utilizado atualmente no contexto de aplicações SPA, onde o backend não gerencia nenhuma sessão do usuário e quando diferentes backends atendem diferentes clientes, extraindo informações sobre o requisitante através do token Não propõe confidencialidade ou criptografia de dados, apenas descreve uma maneira escalável de autenticar requisições.

Keycloak

Solução open source de gerenciamento de acesso e identidade voltada para aplicações e serviços modernos distribuídos. Facilita a configuração de segurança de aplicativos e serviços com pouco ou nenhum código

  • Single-Sign On: Possibilita que o usuário faça login apenas uma vez, e depois acesse todos os sistemas que estejam configurados no Keycloak. O Keycloak também prove a tela de login, e uma maneira facil de configurar o armazenamento e cadastramento de novos usuários em um banco de dados. Também é possível fazer logout, de uma maneira bem simples.
  • Integração com credenciais de redes sociais
  • Pronto p/ escalar horizontalmente (clustering)
  • Integração com LDAP e Active Directory (funcionando na rede do TRT22)
  • Suporta os protocolos JWT, OpenID Connect, OAuth 2.0, SAML.
  • Painel administrativo web
Prática

> curl -v -H "content-type: application/x-www-form-urlencoded" -d 'grant_type=password&client_id=trt22-demo-backend&username=lucas.simao&password=123456' http://keycloak.trt22.jus.br:8080/auth/realms/trt22/protocol/openid-connect/token
